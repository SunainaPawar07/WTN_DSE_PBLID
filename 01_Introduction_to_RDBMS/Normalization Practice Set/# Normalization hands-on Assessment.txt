# Normalization Assessment

## Objective

The objective of this assessment is to practice identifying functional dependencies, determining the current normal form of a relation, identifying normalization issues, and decomposing relations into higher normal forms.

---

# Question 1
Observe the structure as of a given EMPLOYEE table as below and suggest if this table can be further Normalized
EMPNO,ENAME,SAL,DEPTNO,DNAME,LOC
Explain what normal form this table is and how you can make this into next normal form.

## Given Relation

```

EMPLOYEE( EMPNO, ENAME, SAL, DEPTNO, DNAME, LOC)

```

---

## Functional Dependencies

```

EMPNO → ENAME, SAL, DEPTNO
DEPTNO → DNAME, LOC

```

---

## Analysis

### Primary Key

```

EMPNO

```

### Normal Form Check

-  1NF: Yes (All attributes contain atomic values.)
-  2NF: Yes (Primary key consists of a single attribute, so partial dependency does not exist.)
-  3NF: No

### Reason

The relation contains a **Transitive Dependency**.

```

EMPNO → DEPTNO
DEPTNO → DNAME, LOC



Therefore,



EMPNO → DNAME, LOC

```

Since non-key attributes (DNAME and LOC) depend on another non-key attribute (DEPTNO), the relation violates Third Normal Form (3NF).

---

## 3NF Decomposition

### EMPLOYEE

| EMPNO | ENAME | SAL | DEPTNO |
|-------|-------|-----|--------|

### DEPARTMENT

| DEPTNO | DNAME | LOC |
|--------|-------|-----|

---

## Conclusion

The transitive dependency is removed by separating department details into a new table. The resulting relations satisfy Third Normal Form (3NF).

# Question 2
Observe the structure as of a given STUDENT table as below and suggest if this table can be further Normalized
ROLLNO, NAME,AGE,EXAM, MARKS, GRADE
Explain what normal form this table is and how you can make this into next normal form

## Given Relation

```

STUDENT( ROLLNO, NAME, AGE, EXAM, MARKS, GRADE)

```

---

## Assumption

The composite primary key is:

```

(ROLLNO, EXAM)

```

---

## Functional Dependencies

```

ROLLNO → NAME, AGE



(ROLLNO, EXAM) → MARKS



MARKS → GRADE

```

---

## Analysis

### Primary Key

```

(ROLLNO, EXAM)

```

### Normal Form Check

- 1NF: Yes (All attributes contain atomic values.)
- 2NF: No (age and name depend only on roll no. hence partial dependency)

### Reason

The relation contains a **Partial Dependency**.

```

ROLLNO → NAME, AGE

```

Since NAME and AGE depend only on part of the composite key (ROLLNO), the relation violates Second Normal Form (2NF).

Additionally,

```

MARKS → GRADE

```

creates a transitive dependency, which would violate Third Normal Form (3NF).

---

## 2NF Decomposition

### STUDENT

| ROLLNO | NAME | AGE |
|--------|------|-----|

### EXAM_RESULT

| ROLLNO | EXAM | MARKS | GRADE |
|--------|------|-------|-------|

---

## Conclusion

The partial dependency is removed by separating student information from examination details, converting the relation into Second Normal Form (2NF).

# Question 3
Observe the structure of a given EMPLOYEE table and suggest what kind of problem this table has and how to solve the problem
EMPNO,PROJECT_NO,NO_OF_DAYS,CUSTOMERNAME
In the above table EMPNO AND PROJECT_NO both are together a composite primary key

## Given Relation

```

EMPLOYEE_PROJECT( EMPNO, PROJECT_NO, NO_OF_DAYS, CUSTOMERNAME)

```

---

## Given

Composite Primary Key

```

(EMPNO, PROJECT_NO)

```

---

## Functional Dependencies

```

PROJECT_NO → CUSTOMERNAME



(EMPNO, PROJECT_NO) → NO_OF_DAYS

```

---

## Analysis

### Primary Key

```

(EMPNO, PROJECT_NO)

```

### Problem Identified

The relation contains a **Partial Dependency**.

```

PROJECT_NO → CUSTOMERNAME

```

Since CUSTOMERNAME depends only on PROJECT_NO (which is part of the composite primary key), the relation violates Second Normal Form (2NF).

---

## 2NF Decomposition

### PROJECT

| PROJECT_NO | CUSTOMERNAME |
|------------|--------------|

### EMPLOYEE_PROJECT

| EMPNO | PROJECT_NO | NO_OF_DAYS |
|-------|------------|------------|

---

## Conclusion

The partial dependency is removed by separating project information into a new table. The resulting relations satisfy Second Normal Form (2NF).
